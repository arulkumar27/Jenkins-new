# Continuous Delivery Workflow

## What is Continuous Delivery?

Continuous Delivery ensures applications are always ready for deployment.

Deployment still requires manual approval.

## Workflow

Build
↓
Test
↓
Package
↓
Staging Environment
↓
Manual Approval
↓
Production

## Benefits

- Reduced Deployment Risk
- Reliable Releases
- Faster Delivery

## Real-Time Scenario

Banking applications use Continuous Delivery because production deployments require approvals.

## Interview Question

Q: Difference between CI and CD?

Answer:

CI:
Automatically builds and tests code.

CD:
Makes software ready for deployment.
