# Jenkins Shared Libraries

## What are Shared Libraries?

Reusable pipeline code stored centrally.

Instead of writing the same code repeatedly, teams use shared libraries.

## Example

Without Shared Library:

Project A:
- Build Code

Project B:
- Build Code

Project C:
- Build Code

Same code repeated.

## With Shared Library

Common functions stored once.

All projects use the same library.

## Benefits

- Reusability
- Standardization
- Easier Maintenance

## Real-Time Scenario

Company has 100 applications.

Common deployment logic stored in a shared library.

All applications use the same deployment function.

## Interview Question

Q: Why are Shared Libraries used?

Answer:
Shared Libraries help reuse common pipeline code across multiple Jenkins projects.
