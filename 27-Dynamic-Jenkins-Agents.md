# Dynamic Jenkins Agents

## What are Dynamic Agents?

Temporary agents created when needed.

Removed after build completion.

## Traditional Model

Fixed Agents

Problems:

- Idle Resources
- Higher Cost

## Dynamic Model

Build Request
↓
Agent Created
↓
Build Executes
↓
Agent Deleted

## Benefits

- Cost Saving
- Auto Scaling
- Better Resource Utilization

## Real-Time Scenario

E-commerce company runs thousands of builds daily.

Agents are created only when required.

## Interview Question

Q: What is a Dynamic Agent?

Answer:
A temporary build node created automatically for executing a Jenkins job.
